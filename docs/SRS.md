# 오이코스 (Oikos) — 요구사항 명세서 (SRS)

문서 버전 1.0 · 2026-07-23 · 앱 버전 0.1.0 기준

> 이 문서가 요구사항 번호(FR/NFR)의 정본이다. 커밋 메시지·코드 주석의 `FR-xxx`는 이 문서를 가리킨다.
> 기존 커밋에서 이미 쓰인 번호(FR-101~105, 208, 402~406, 501~504, 601, NFR-02)는 그대로 유지했고,
> 번호가 없던 기능에는 이 문서에서 빈 구간을 채워 새로 부여했다.

---

## 1. 개요

### 1.1 목적

오이코스는 **소비 습관 형성이 필요한 사람**(가계부 무경험자, 시작해도 며칠 만에 그만뒀던 사람)을 타겟으로,
그들의 소비 습관이 자리 잡는 **기초(base)가 되는 가계부**다.

핵심 철학: *"돈을 관리하게 만드는 것이 아니라, 자연스럽게 이해하게 만든다 — 이해가 쌓이면 습관이 된다."*

습관 형성의 수단은 게임화(스트릭·배지·리마인더)가 아니라 **낮은 마찰(3초 기록)과 담백한 피드백(인사이트 한 문장)**이다.

### 1.2 범위

- Flutter 앱 — Android / iOS / Web 단일 코드베이스
- 배포: Google Play·App Store(준비 중), 웹 데모 <https://yscof.github.io/oikos/app/>
- 기본 동작은 **완전 오프라인**(기기 내 저장). Supabase URL/키가 주입된 빌드에서만 로그인·클라우드가 켜진다(config-gate)

### 1.3 용어

| 용어 | 정의 |
|---|---|
| 기록(Entry) | 지출 또는 수입 1건. 금액·카테고리·메모·감정·발생시각 포함 |
| 인사이트 | 룰 엔진이 만든 사용자 노출 문장. 홈 헤드라인 + 보조 라인 |
| 정산월 | 가계부 시작일(FR-601)부터 다음 시작일 전날까지의 "이번 달" 구간 |
| 톤 계약 | 사용자 노출 문장에 명령형·판단어를 금지하는 규칙. 테스트로 기계 검증 |
| config-gate | 빌드 시 `--dart-define`으로 Supabase 값이 주입됐을 때만 계정 기능 활성화 |

## 2. 제품 원칙 (모든 요구사항에 우선)

1. 첫 화면은 거래내역이 아니라 **"내 금융 상태" 한 문장** — 날씨앱이 날씨를 보여주듯
2. 기록은 **평균 3초**: 금액 입력 → 스마트 카테고리 추천 → 저장
3. **Invisible AI** — 룰 기반 온디바이스 인사이트. "AI"라는 단어는 앱 어디에도 없음
4. **게임화 금지** — 포인트·배지·스트릭 없음. 알림 남발 없음. 지출에 빨간 숫자 없음
5. 편안함이 기본값 — 편안해야 습관이 오래 간다

## 3. 기능 요구사항 (FR)

상태: ✅ 구현됨 · ⚠ 부분 구현 · ⬜ 예정

### FR-1xx 계정 (config-gate — 미설정 빌드는 로그인 화면 없이 완전 오프라인)

| 번호 | 요구사항 | 상태 |
|---|---|---|
| FR-101 | 이메일/비밀번호로 가입할 수 있다 | ✅ |
| FR-102 | 이메일/비밀번호로 로그인·로그아웃할 수 있고 세션이 유지된다 | ✅ |
| FR-103 | 비밀번호를 잊으면 이메일로 재설정할 수 있다 | ✅ |
| FR-104 | 소셜 로그인(구글/카카오)으로 가입·로그인할 수 있다 | ⬜ |
| FR-105 | 회원 탈퇴 시 계정과 서버의 모든 데이터가 삭제된다 (Edge Function 필요) | ⬜ |
| FR-106 | 기록이 클라우드에 동기화된다 (local-first 양방향, transactions 테이블) | ⬜ |

### FR-2xx 기록

| 번호 | 요구사항 | 상태 |
|---|---|---|
| FR-201 | 지출/수입을 금액·카테고리로 기록한다. 해피패스는 입력 2번 ≈ 3초 | ✅ |
| FR-202 | 카테고리 칩이 시간대·금액대·요일 이력 기반 추천순으로 실시간 재정렬되고 1위가 자동 선택된다 | ✅ |
| FR-203 | 메모("어떤 순간이었나요?")를 선택 입력할 수 있다 | ✅ |
| FR-204 | 발생 날짜를 바꿀 수 있다 (기본: 오늘) | ✅ |
| FR-205 | 지출에 감정 태그를 선택 입력할 수 있다 (수입에는 없음) | ✅ |
| FR-206 | 기록을 수정할 수 있다 (내역에서 탭) | ✅ |
| FR-207 | 기록을 삭제할 수 있다 (내역에서 롱프레스) | ✅ |
| FR-208 | 금액 입력에 천 단위 콤마가 실시간으로 붙는다 | ✅ |

### FR-3xx 홈(가계부) · 인사이트

| 번호 | 요구사항 | 상태 |
|---|---|---|
| FR-301 | 홈 상단에 룰 기반 헤드라인 인사이트 한 문장을 보여준다. 같은 날 같은 데이터면 항상 같은 문장(결정적) | ✅ |
| FR-302 | 헤드라인을 탭하면 "이렇게 읽었어요" 근거 수치 시트를 보여준다 (설명가능성) | ✅ |
| FR-303 | 보조 감각 라인(≤2개)을 헤드라인 아래에 보여준다 | ✅ |
| FR-304 | "오늘 지출 N원" / "이번 주 지출 N원" 소문을 무채색으로 보여준다 | ✅ |
| FR-305 | 최근 기록 3건을 보여준다 | ✅ |
| FR-306 | 누적 기록 수("지금까지 N번 기록했어요")를 보여준다 — 스트릭 아님, 기록이 없으면 숨김 | ✅ |
| FR-307 | 기록이 적으면(8건 미만 또는 7일 미만) 콜드스타트 문장을 보여준다 | ✅ |

### FR-4xx 조회 · 통계

| 번호 | 요구사항 | 상태 |
|---|---|---|
| FR-401 | 내역을 날짜 그룹 타임라인(오늘/어제/M월 d일)으로 보여준다. 월 경계에 "N월 · 지출 N원" 구분선 | ✅ |
| FR-402 | 내역을 캘린더(월 그리드)로 볼 수 있다 | ✅ |
| FR-403 | 홈에서 이번 달 지출/수입/남은 금액 요약을 보여준다 (정산월 기준) | ✅ |
| FR-404 | 통계 탭에서 이번 달 카테고리별 지출 비율을 가로 막대로 보여준다 | ✅ |
| FR-405 | 내역에서 목록 ↔ 캘린더 뷰를 토글할 수 있다 | ✅ |
| FR-406 | 내역을 키워드(메모)와 카테고리로 검색·필터할 수 있다 | ✅ |

### FR-5xx 예산

| 번호 | 요구사항 | 상태 |
|---|---|---|
| FR-501 | 월 전체 예산을 설정할 수 있다. 0이면 미설정으로 취급해 예산 UI를 숨긴다 | ✅ |
| FR-502 | 홈에서 예산 대비 지출 진행률을 보여준다 | ✅ |
| FR-503 | 카테고리별 예산을 설정할 수 있다 | ⬜ |
| FR-504 | 예산 80%/100% 도달 시 담백한 인앱 알림을 보여준다 (푸시 아님) | ✅ |

### FR-6xx 설정 · 더보기

| 번호 | 요구사항 | 상태 |
|---|---|---|
| FR-601 | 가계부 시작일(정산 기준일, 매월 1~28일)을 설정할 수 있다. 모든 "이번 달" 계산이 이를 따른다 | ✅ |
| FR-602 | 화면 모드(시스템 기본/라이트/다크)를 선택할 수 있고 재실행 후에도 유지된다 | ✅ |
| FR-603 | 모든 기록을 JSON으로 내보낼 수 있다 (클립보드 복사) | ✅ |
| FR-604 | 모든 데이터를 확인 후 삭제할 수 있다 | ✅ |
| FR-605 | 오픈소스 라이선스와 앱 버전을 확인할 수 있다 | ✅ |

### FR-7xx 내비게이션 · 화면 구조

| 번호 | 요구사항 | 상태 |
|---|---|---|
| FR-701 | 하단 탭 3개: 왼쪽부터 **가계부 · 통계 · 더보기**. 앱 실행 시 첫 화면은 가계부 | ✅ |
| FR-702 | 더보기 탭에 화면 모드(FR-602)와 설정 진입 항목이 있다 | ✅ |
| FR-703 | 내역·설정은 탭 위로 push되는 상세 화면이다. 탭 전환 시 각 탭의 상태가 유지된다 | ✅ |
| FR-704 | 웹에서 화면이 넓으면(>600px) 앱을 폰 폭(430px)으로 가운데 정렬해 보여준다 | ✅ |

## 4. 비기능 요구사항 (NFR)

| 번호 | 요구사항 |
|---|---|
| NFR-01 | **오프라인 우선.** 계정 미설정 빌드는 서버·수집 없이 모든 기능이 기기 내에서 동작한다 |
| NFR-02 | **본인 데이터만.** 클라우드 사용 시 RLS로 본인 행만 읽고 쓸 수 있다 |
| NFR-03 | **성능.** 기록 해피패스 평균 3초. 헤비유저(연 ~3.6천 건)에서도 로컬 저장(SharedPreferences JSON)이 감당 — 1만 건 초과 시 sqflite 전환 계획만 문서화 |
| NFR-04 | **톤 계약.** 사용자 노출 인사이트 문장은 `lib/insight/insight_messages.dart` 한 곳에만 존재하고, 명령형(~하세요)·판단어(과소비/낭비)를 쓰지 않는다. `test/tone_test.dart`가 기계 검증 |
| NFR-05 | **결정성.** 인사이트는 같은 날 같은 데이터면 항상 같은 문장 (신뢰) |
| NFR-06 | **금액 표기.** 모든 지출·수입·요약 금액은 천 단위 콤마 표기(예: 124,000원). 만/천 축약(12만 4천원)은 쓰지 않는다. 예외: 인사이트 근거 문장의 어림 표현(3.2만원) |
| NFR-07 | **시각.** 지출 금액에 빨간색 금지. 시드 컬러 `#5E7A66`(세이지 그린), 라이트/다크 모두 지원 |
| NFR-08 | **개인정보.** 오프라인 빌드는 "모든 데이터는 이 기기에만 저장됩니다"가 사실이어야 한다. 계정 빌드 배포 전 privacy.html·설정 문구 갱신 필수 |

## 5. 개발 제약사항

- codegen(build_runner) 의존성 금지
- 저장 키 `oikos_entries_v1` — 포맷 변경 시 v2 키 + 마이그레이션 필수
- 스택: Flutter + Riverpod + go_router + shared_preferences. 차트·캘린더 외부 패키지 없이 순수 위젯
- main 푸시마다 CI(analyze → test → APK)와 웹 배포(GitHub Pages)가 실행된다
- 스토어 업로드 시 `pubspec.yaml`의 buildNumber(+N) 증가
