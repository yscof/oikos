# 1. 프로젝트 개요

## 프로젝트명

- 오이코스 (Oikos) — 그리스어 οἶκος(집·가정, economy의 어원)에서 이름을 딴 가계부 앱

## 프로젝트 목적

- 소비 습관 형성이 필요한 사람(가계부 무경험자, 며칠 만에 포기해본 사람)에게 **소비 습관이 자리 잡는 기초(base)가 되는 가계부**를 제공한다.
- 핵심 철학: "돈을 관리하게 만드는 것이 아니라, 자연스럽게 이해하게 만든다 — 이해가 쌓이면 습관이 된다."

## 프로젝트 배경

- 기존 가계부는 기록 부담(입력 항목 많음)과 압박(빨간 숫자, 과소비 경고, 스트릭)으로 초심자가 며칠 만에 포기한다.
- 오이코스는 습관 형성의 수단을 게임화가 아니라 **낮은 마찰(3초 기록)과 담백한 피드백(룰 기반 인사이트 한 문장)**에 둔다.
- 게임화 금지(포인트·배지·스트릭 없음), 알림 남발 없음, 지출에 빨간 숫자 없음 — 편안함이 기본값.

## 시스템 범위

- Flutter 단일 코드베이스: Android / iOS / Web
- 기본 **완전 오프라인**(모든 데이터 기기 내 저장). Supabase URL/키가 주입된 빌드에서만 로그인·클라우드 활성화(config-gate)
- 하단 탭 3개(가계부 · 통계 · 더보기) + 상세 화면(내역, 설정)
- 배포: Google Play·App Store(준비 중), 웹 데모 <https://yscof.github.io/oikos/app/>
- v1 제외: 계좌·카드 연동, 푸시 알림, 반복 거래, 다중 통화, 위젯

## 고객(발주처)

- 개인 프로젝트 (자체 서비스)

## 수행사

- yscof (GitHub: [yscof/oikos](https://github.com/yscof/oikos))

# 2. 용어 사전

| 용어 | 정의 | 비고 |
|------|------|------|
| 사용자 | 앱을 설치해 소비를 기록하는 개인. 계정 없이도 전 기능 사용 가능 | 타겟: 소비 습관 형성이 필요한 사람 |
| 소비 기록(Entry) | 지출 또는 수입 1건. 금액·카테고리·메모·감정·발생시각 포함 | 저장 키 `oikos_entries_v1` |
| 인사이트 | 룰 엔진이 생성하는 사용자 노출 문장(홈 헤드라인 + 보조 라인) | "AI"라는 단어는 앱 어디에도 없음 (Invisible AI) |
| 정산월 | 가계부 시작일부터 다음 시작일 전날까지의 "이번 달" 구간 | 기본 매월 1일, 1~28일 설정 가능 |
| 톤 계약 | 사용자 노출 문장에 명령형(~하세요)·판단어(과소비/낭비)를 금지하는 규칙 | `test/tone_test.dart`가 기계 검증 |
| config-gate | 빌드 시 `--dart-define`으로 Supabase 값이 주입된 경우에만 계정 기능이 켜지는 구조 | 미주입 빌드 = 완전 오프라인 |

## MAIN01 소비 기록

### MAIN01_RECORD01 소비 등록

**상세 설명**

- 사용자는 지출/수입을 기록할 수 있다. 해피패스는 입력 2번 ≈ **3초**: 금액 입력 → 카테고리 자동 선택 → 저장.
- 기록 시트가 열리면 금액 필드에 자동 포커스(숫자 키보드)되고, 금액 입력 중 천 단위 콤마가 실시간으로 붙는다.
- 지출/수입 세그먼트(지출 기본), 발생 날짜는 기본 "오늘"이며 변경 가능.
- 저장 후 팡파레·축하 연출 없음(게임화 금지).

**필수 데이터**

- 금액(원)
- 종류(지출/수입)
- 카테고리(추천 1위 자동 선택)

**선택 데이터**

- 메모("어떤 순간이었나요?")
- 감정 태그(지출만, 수입에는 없음)
- 발생 날짜(기본 오늘)

**우선순위**

- 필수

**비고**

- 구현됨 (구 FR-201·203·204·205·208)

## MAIN02 소비 분석 (인사이트)

### MAIN02_ANALYSIS01 헤드라인 인사이트

**상세 설명**

- 홈 첫 화면에 "내 금융 상태"를 한 문장으로 보여준다(거래내역 표 아님).
- 룰 기반 엔진(카테고리 증가/감소, 빈도, 조용한 주, 요일 패턴, 콜드스타트 + steady 폴백)이 우선순위로 선택한다.
- 같은 날 같은 데이터면 항상 같은 문장(결정적).
- 기록이 적으면(8건 미만 또는 7일 미만) 콜드스타트 문장을 보여준다: "기록이 쌓이면 소비의 흐름을 읽어드릴게요."
- 모든 문장은 톤 계약을 지킨다(명령형·판단어 금지).

**필수 데이터**

- 소비 기록 목록

# 4. 비기능 요구사항

## 성능

### NFR01 응답 속도

- 기록 해피패스 평균 3초 이내(금액 입력 → 저장)
- 모든 계산이 온디바이스 — 네트워크 지연 없음
- 헤비유저(연 ~3.6천 건)에서도 로컬 저장(SharedPreferences JSON)이 감당. 1만 건 초과 시 sqflite 전환 계획 문서화

---

## 보안·개인정보

### NFR02 데이터 보호

- 오프라인 빌드: 서버·계정·수집 없음. "모든 데이터는 이 기기에만 저장됩니다"가 사실이어야 한다
- 계정 빌드: HTTPS(Supabase) 통신, RLS로 본인 행만 접근(구 NFR-02), anon 키만 클라이언트 노출
- 계정 빌드 배포 전 privacy.html·설정 문구 갱신 필수

---

## 호환성

### NFR03 지원 환경

- Android / iOS / Web(Chrome, Safari, Edge)
- 라이트/다크 모드 모두 지원

---

## 확장성

### NFR04 구조 확장

- 인사이트는 룰 기반 온디바이스를 유지한다(외부 LLM 교체 대상 아님 — Invisible AI 철학)
- 저장소는 프로바이더 경유 단일 진입 — SharedPreferences → sqflite 교체 시 1파일 수정
- 인사이트 룰은 pure function으로 추가·제거 용이

---

## 유지보수

### NFR05 품질 관리

- main 푸시마다 CI: analyze → test → APK 빌드, 웹은 GitHub Pages 자동 배포
- 사용자 노출 인사이트 문장은 `lib/insight/insight_messages.dart` 한 곳에만 — 톤 계약을 테스트가 기계 검증
- 인사이트 결정성(같은 날 같은 데이터 = 같은 문장)을 테스트로 보증
- codegen(build_runner) 의존성 금지

---

## 표기·UX 원칙

### NFR06 화면 표기

- 지출·수입·요약 금액은 천 단위 콤마 표기(예: 124,000원). 만/천 축약 금지 (예외: 인사이트 근거 문장의 어림 표현)
- 지출 금액에 빨간색 금지. 시드 컬러 `#5E7A66`(세이지 그린)
- 게임화 요소(포인트·배지·스트릭) 금지, "AI" 단어 노출 금지
